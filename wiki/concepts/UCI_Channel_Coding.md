# UCI_Channel_Coding

## 정의
[[UCI]] (Uplink Control Information) 채널 코딩은 전송되는 제어 정보의 신뢰성을 확보하기 위해 수행되는 과정으로, 정보 비트에 대한 [[CRC]] 부착, 코드 블록 세그멘테이션, 그리고 [[Polar]] 코드 또는 소형 블록(Small block) 코드를 이용한 부호화 절차를 의미합니다.

## 요약
[[UCI]] 채널 코딩은 정보 비트의 길이에 따라 두 가지 경로로 나뉩니다. 정보 비트가 특정 임계값보다 크거나 같을 경우 [[Polar]] 코드를 사용하며, 그보다 작을 경우 소형 블록 코드를 사용하여 부호화를 수행합니다. 이 과정에서 오류 검출을 위한 [[CRC]]가 부착되며, 필요 시 코드 블록 세그멘테이션이 수행됩니다.

## 상세 설명
[[UCI]] 채널 코딩 절차는 TS 38.212에 정의된 바와 같이 다음 단계로 구성됩니다.

1. **코드 블록 세그멘테이션 및 CRC 부착**: 
   - 입력된 [[UCI]] 비트 시퀀스에 대해 오류 검출을 위한 [[CRC]]를 계산하여 부착합니다.
   - 정보 비트의 크기가 특정 임계값을 초과하는 경우, 전체 비트 시퀀스를 여러 개의 코드 블록으로 분할하는 세그멘테이션 과정을 거칩니다.
2. **채널 코딩**:
   - **[[Polar]] 코드**: 정보 비트의 양이 상대적으로 많은 경우 적용됩니다. [[Polar]] 코딩은 채널의 신뢰도를 높이기 위해 비트들을 재배치하고 부호화하는 방식을 취합니다.
   - **소형 블록 코드 (Small block codes)**: 정보 비트의 양이 적은 경우(예: 11비트 이하)에 적용됩니다. 이 방식은 복잡도가 낮은 효율적인 부호화 알고리즘을 사용하여 제어 정보를 보호합니다.
